🧭 GitBranchSwitcher

GitBranchSwitcher 是一个基于 WinForms (.NET 6+) 的多仓库 Git 分支管理工具，支持自动检测、并发切换、Stash/强制模式、状态图显示等功能。
适用于游戏、引擎、工具链等多 Git 仓库协同开发场景。

✨ 功能概览
功能	描述
🧩 多仓库管理	自动加载指定父目录下的多个 Git 仓库，显示当前分支与状态
🔄 一键切线	批量切换分支，支持 Stash 改动或强制丢弃改动
⚙️ 并发执行	同时最多支持 4 个仓库并行切线，加速操作
💾 Stash 记忆	“切线时贮存改动”选项支持自动记忆
📌 使用当前分支	自动填入任一选中仓库的当前分支名到目标框
🪶 状态图显示	图形化显示当前状态（idle / 切线中 / 成功 / 失败）
🪟 内嵌图标	AppIcon.ico 已作为嵌入资源，无需额外图标文件
🧰 可视化日志	实时日志显示每个仓库的执行状态与错误信息
🖥️ 使用方式

打开程序后，点击 “选择父目录”，选择包含多个 Git 仓库的根目录。
工具会自动扫描所有子文件夹并识别 Git 仓库。

勾选需要操作的仓库。

在目标分支框中输入或选择要切换的目标分支。

也可点击“使用当前分支”快速填入。

删除全部文本时不会再崩溃（已修复安全逻辑）。

选择模式：

强制切线（丢弃改动）

Stash 切线（自动保存改动）

点击“一键切线”：

工具会并行切换分支（默认同时 4 个任务）；

日志区实时输出进度；

状态图会随状态更新（idle → 切线中 → 完成/失败）。

🧠 状态图规则

状态图显示逻辑由 StatusImageService.cs 管理：

状态关键字	对应目录	示例图
idle	./Assets/state_idle/	空闲状态
切线中	./Assets/state_switching/	正在切线
切线完成	./Assets/state_done/	成功完成
切线失败	./Assets/state_failed/	失败提示

若找不到图片，会自动生成灰底占位图并记录 resource_dump.txt，用于调试内嵌资源路径。
